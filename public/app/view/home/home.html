Overall functionality:

This projects is a combination of a database, node-js server with rest-api and a client-end angular application. The purpose of it is to correctly handle large amounts of data and serve the relevant content to the webpage.

Functionality on the webpage consists of:

"Home" view shows the intro text and this documentation.

"Search Wikis" view, which allows the user to search for matching titles in the database and by clicking a result is directed to a wiki page on the subject.

"Search Wikis-2" view, which does the same as the first one, except the results are now paginated and shown in accordion, which when clicked, expands into a wiki page.

"Categories" view allows the user to click a letter which results in a sorted paginated view including all categories beginning with that letter. When a category is clicked, the browser is directed to a new view showing the topics belonging to that category. When a topic is then clicked, the corresponding wiki page is shown.

"Categories-2" view is similar to the "Categories view" except all the categories are now shown in accordions. They expand to show topics in the corresponding category, which when clicked, redirects to the wiki page.

Backend tests:

We set up a test database with some test data, and then tested each one of the four different routes in the rest-api, checking if the returned data matches the expected value.

Frontend tests:

We did a test of a factory and if its methods where defined. Then we proceeded to test the individual methods and their http requests, while using $httpBackend to see if the correct value came back.
